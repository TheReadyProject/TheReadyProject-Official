Curso de SQL

1. SQL

Módulo 1: Introdução ao SQL

Aula 1: Conceitos Básicos

Objetivo: Introdução aos conceitos básicos de SQL.

Conteúdo:

O que é SQL:

SQL (Structured Query Language) é uma linguagem de consulta estruturada usada para gerenciar bancos de dados relacionais. Com SQL, podemos criar bancos de dados, tabelas, executar consultas, inserir, atualizar e excluir dados.

Comandos SQL:

  • SELECT: Seleciona dados de um banco de dados.
  • INSERT INTO: Insere novos registros em uma tabela.
  • UPDATE: Atualiza registros existentes em uma tabela.
  • DELETE FROM: Exclui registros de uma tabela.
  • CREATE DATABASE: Cria um novo banco de dados.
  • CREATE TABLE: Cria uma nova tabela em um banco de dados.
  • DROP DATABASE: Exclui um banco de dados existente.
  • DROP TABLE: Exclui uma tabela existente.

Aula 2: Consultas Básicas

Objetivo: Aprender a executar consultas simples em SQL.

Conteúdo:

Consulta de dados:

SELECT * FROM tabela;

Filtragem de resultados:

SELECT * FROM tabela WHERE condicao;

Ordenação de resultados:

SELECT * FROM tabela ORDER BY coluna ASC|DESC;

Limitando resultados:

SELECT * FROM tabela LIMIT quantidade;

Aula 3: Operações de CRUD

Objetivo: Aprender a realizar operações básicas de CRUD (Create, Read, Update, Delete) em SQL.

Conteúdo:

Inserção de dados:

INSERT INTO tabela (coluna1, coluna2) VALUES (valor1, valor2);

Atualização de dados:

UPDATE tabela SET coluna = novo_valor WHERE condicao;

Exclusão de dados:

DELETE FROM tabela WHERE condicao;

2. Prática com Bancos de Dados

Módulo 2: Aplicação de Conceitos

Aula 1: Modelagem de Dados

Objetivo: Aprender a modelar bancos de dados utilizando diagramas entidade-relacionamento (DER).

Conteúdo:

Entidades e atributos:

Entidades representam objetos do mundo real, enquanto os atributos são características desses objetos.

Relacionamentos:

Relacionamentos definem como as entidades estão conectadas entre si.

Diagramas DER:

Os diagramas entidade-relacionamento são ferramentas visuais usadas para representar a estrutura do banco de dados.

Aula 2: Normalização de Dados

Objetivo: Compreender os conceitos de normalização de dados e aplicá-los na modelagem de bancos de dados.

Conteúdo:

Primeira Forma Normal (1FN):

Elimina a redundância de dados, garantindo que cada coluna contenha apenas um valor.

Segunda Forma Normal (2FN):

Atende à 1FN e garante que cada coluna seja totalmente dependente da chave primária.

Terceira Forma Normal (3FN):

Atende à 2FN e elimina dependências transitivas, garantindo que cada coluna seja dependente apenas da chave primária.

Aula 3: Consultas Avançadas

Objetivo: Aprender consultas SQL mais avançadas para extrair informações específicas de um banco de dados.

Conteúdo:

Junção de Tabelas (JOIN):

SELECT coluna1, coluna2 FROM tabela1 JOIN tabela2 ON tabela1.coluna = tabela2.coluna;

Subconsultas (Subqueries):

SELECT coluna FROM tabela WHERE coluna IN (SELECT coluna FROM outra_tabela);

Funções de Agregação:

SELECT COUNT(coluna) FROM tabela;

3. Práticas Avançadas

Módulo 3: Otimização de Desempenho

Aula 1: Índices e Otimização

Objetivo: Compreender a importância dos índices na otimização do desempenho de consultas.

Conteúdo:

Criação de Índices:

CREATE INDEX index_name ON tabela (coluna);

Tipo de Índices:

Índices podem ser criados em uma ou várias colunas e podem ser únicos ou não.

Aula 2: Transações e Controle de Concorrência

Objetivo: Entender o conceito de transações em bancos de dados e como garantir a consistência dos dados em ambientes concorrentes.

Conteúdo:

Transações:

Uma transação é uma sequência de operações que formam uma unidade de trabalho indivisível.

Propriedades ACID:

  • Atomicidade: Todas as operações de uma transação são realizadas com sucesso ou nenhuma é realizada.
  • Consistência: O banco de dados permanece em um estado consistente antes e depois da transação.
  • Isolamento: As operações de uma transação são isoladas de outras transações concorrentes.
  • Durabilidade: As alterações feitas por uma transação persistem, mesmo em caso de falha do sistema.

Aula 3: Segurança e Backup

Objetivo: Aprender a proteger os dados do banco de dados e realizar backups para garantir a sua integridade.

Conteúdo:

Controle de Acesso:

Restringir o acesso ao banco de dados apenas a usuários autorizados.

Backup e Recuperação:

Realizar backups regulares dos dados para evitar perdas em caso de falha do sistema.
